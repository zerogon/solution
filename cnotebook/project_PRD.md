# PRD – 작가용 캐릭터 관리 서비스

## 1. 제품 개요

### 제품명 (가칭)

Character Notebook

### 제품 목적

소설 및 웹소설 작가가 **작품별 캐릭터 정보를 체계적으로 관리하고 원고 작성 중 빠르게 조회**할 수 있도록 지원하는 서비스.

### 핵심 가치

1. 작품별 캐릭터 분리 관리
2. 캐릭터 설정 빠른 검색
3. 캐릭터 그룹 관리 (폴더)
4. 이미지 기반 캐릭터 확인

---

# 2. 핵심 사용자

### Primary User

* 웹소설 작가
* 소설 작가
* 시나리오 작가

### 주요 사용 시나리오

1. 작가가 새 작품을 생성
2. 캐릭터 정보를 등록
3. 캐릭터 이미지와 설정 관리
4. 원고 작성 중 캐릭터 검색
5. 캐릭터 그룹 (폴더) 관리

---

# 3. 서비스 구조

서비스 구조는 **작품 단위 관리 시스템**이다.

```
작품
 ├ 캐릭터
 │   ├ 캐릭터 정보
 │   └ 이미지
 │
 └ 폴더
     └ 캐릭터 그룹
```

핵심 규칙

* 캐릭터는 반드시 **작품에 종속**
* 다른 작품 캐릭터와 **절대 섞이지 않음**

# 8. API 구조

### 작품

생성

```
POST /api/work
```

조회

```
GET /api/work
```

---

### 캐릭터

생성

```
POST /api/character
```

목록

```
GET /api/character?workId=
```

상세

```
GET /api/character/{id}
```

수정

```
PATCH /api/character/{id}
```

삭제

```
DELETE /api/character/{id}
```

---

### 폴더

생성

```
POST /api/folder
```

폴더 캐릭터 추가

```
POST /api/folder/add-character
```
