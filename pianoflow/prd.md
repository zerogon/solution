# 피아노 학원 레슨 예약 시스템 PRD

## 프로젝트 개요

### 프로젝트명

Piano Lesson Booking System (PianoFlow)

### 목표

약 50명의 레슨생과 3명의 피아노 선생님이 효율적으로 레슨 예약 및 관리를 할 수 있는 PWA 기반 예약 시스템 구축.

### 핵심 가치

* 학원 운영자의 예약/회원 관리 효율화
* 레슨생의 간편한 모바일 예약 경험
* 선생님의 직관적인 일정 관리
* 모바일 앱 수준의 UX 제공

---

# 1. 사용자 유형 및 권한

## 1.1 관리자(Admin)

### 권한

* 전체 사용자 관리
* 선생님 권한 부여/해제
* 예약 전체 조회 및 수정
* 레슨 횟수 관리
* 휴면 회원 관리
* 선생님별 스케줄 설정

### 주요 기능

* 회원 등록/수정/삭제
* 선생님 가능 요일 설정
* 예약 강제 생성/취소
* 전체 캘린더 조회

---

## 1.2 선생님(Teacher)

### 권한

* 본인 레슨 일정 조회
* 본인 담당 학생 확인
* 레슨 일정 상태 확인

### 제한사항

* 별도 버튼으로 다른 선생님 일정 확인 가능하게 구현
* 회원 정보 수정 불가
* 시스템 설정 변경 불가

---

## 1.3 레슨생(Student)

### 권한

* 본인 레슨 예약
* 본인 예약 취소
* 남은 레슨 횟수 조회
* 본인 예약 내역 조회

### 제한사항

* 본인 예약만 접근 가능
* 당일 취소 불가
* 타 사용자 정보 접근 불가

---

# 2. 핵심 기능 요구사항

# 2.1 인증 및 로그인

## 로그인 방식

* 사용자 ID: 휴대폰 번호 뒷 4자리


## 추가 고려사항

* 휴대폰 번호 중복 검증 필요
* 동일 뒷자리 충돌 대응 필요

### 권장 방식

```text
로그인 ID:
010-1234-5678 → 5678a
010-9876-5678 → 5678b
```

### 인증 기술

* NextAuth.js 또는 Auth.js 사용 권장
* JWT Session 기반 인증

---

# 2.2 레슨 예약 시스템

## 운영 시간

* 오전 10시 ~ 오후 10시
* 1시간 단위 예약

### 예약 가능 시간 예시

* 10:00
* 11:00
* ...
* 22:00

(23:00 종료)

---

## 예약 규칙

### 예약 가능 조건

* 학생의 남은 레슨 횟수 > 0
* 선생님의 가능 요일
* 이미 예약되지 않은 시간
* 본인 중복 예약 불가

### 취소 규칙

* 전날 23:59까지 취소 가능
* 당일 취소 불가

### 예시

* 5월 20일 레슨
  → 5월 19일 23:59까지만 취소 가능
