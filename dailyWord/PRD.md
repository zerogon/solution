
# PRD — 카페 QR 이벤트 서비스 “오늘의 단어”

## 1. 제품 개요

카페 방문 고객이 QR 코드를 스캔하여 웹 페이지에 접속한 뒤,
3×3 단어 그리드에서 하나의 긍정적인 단어를 선택하면 해당 단어와 관련된 **따뜻한 문장 또는 운세 메시지**를 확인할 수 있는 참여형 이벤트 서비스이다.

서비스는 **QR 기반 유입 → 단어 선택 → 결과 확인 → 오늘의 단어 통계 확인** 흐름으로 구성된다.

또한 운영자는 다음 데이터를 확인할 수 있다.

* QR 페이지 방문자 수
* 실제 단어 선택 참여 수
* 단어별 선택 통계

이를 통해 **QR 유입 대비 참여율**을 측정할 수 있다.

# 3. 핵심 기능

## 3.1 QR 기반 페이지 진입

사용자는 카페에 비치된 **QR 코드**를 통해 웹 페이지에 접속한다.

### 요구사항

* 로그인 없이 접근 가능
* 모바일 최적화 UI
* 접속 시 **방문 로그 기록**

### 기록 데이터

* 방문 시간
* session id
* user agent

# 3.2 단어 선택 화면

### UI 구조

3×3 격자 구조

```
행복     자유     성장
희망     용기     사랑
도전     발전     ?
```

구성

* 8칸: 긍정적인 단어

### 랜덤 선택 버튼

UI 특징

* `?` 표시
* 배경에 흐릿하게 **랜덤 선택 텍스트**

# 3.3 단어 선택 애니메이션

# 4. 운영 통계 기능

운영자가 확인 가능한 핵심 지표

## 4.1 QR 방문자 수

QR을 통해 웹 페이지에 접속한 사용자 수

정의

```
QR 페이지 최초 로딩 발생 횟수
```

집계

```
daily_visit_count
```

---

## 4.2 단어 선택 참여 수

단어를 실제로 선택한 사용자 수

정의

```
단어 선택 이벤트 발생 횟수
```

집계

```
daily_selection_count
```

---

## 4.3 참여율

QR 방문 대비 실제 참여율

공식

```
participation_rate
= selection_count / visit_count
```

예시

```
방문자 120명
참여자 90명

참여율 75%
```

---

## 4.4 단어별 선택 통계

단어별 선택 횟수

예시

```
행복 34회
자유 21회
성장 15회
```

# 8. 통계 집계 방식

## 방문자 수

```
SELECT COUNT(*)
FROM visits
WHERE visited_at >= today
```

---

## 참여자 수

```
SELECT COUNT(*)
FROM selections
WHERE selected_at >= today
```

---

## 단어별 통계

```
SELECT word_id, COUNT(*)
FROM selections
WHERE selected_at >= today
GROUP BY word_id
```

---

## 퍼센트 계산

```
word_count / total_selection * 100
```
